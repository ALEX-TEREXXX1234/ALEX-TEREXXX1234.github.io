Рис. 1. Представление в NetMaker
Обратите внимание, что NetMaker автоматически распознал, что первая строка файла исходных данных — не значения, а названия параметров.
Необходимо предсказать значение среднегодовой температуры на год вперед, поэтому каждый пример для обучения нейросети должен включать значение МАТ будущего года. Это означает, что каждому входному вектору надо поставить в соответствие будущее значение МАТ. NetMaker предполагает, что в каждой строке таблицы находится один пример для обучения сети. Поэтому необходимо включить значение МАТ будущего года во все примеры.
Добавьте к исходным данным столбец, который содержал бы те же значения, что и колонка МАТ, но был бы смещен на 1 ячейку вверх.
Для этого:
> в меню Column (операции со столбцами) выполните команду Shift Column Up (сдвинуть столбец вверх);
> выберите мышью столбец МАТ;
> введите 1 в ответ на вопрос о количестве ячеек, на которое сдвигается столбец (Number of rows to shift up);
> в окне Column Name (имя столбца) введите имя нового столбца,например, МАТ+1.
Обратите внимание, что теперь значению МАТ за год п в новом столбце соответствует значение температуры за год n+1. Последняя ячейка столбца МАТ+ 1 содержит некорректное значение. Программе не было известно значение МАТ за 1986 год, поэтому она поместила туда данные 1985 года. Эту строку нельзя будет использовать при создании базы примеров — сеть должна обучаться на действительных данных.
Данные входного вектора должны содержать несколько предыдущих значений прогнозируемой величины. Ограничимся одним — прошлогодним значением температуры.
Создайте новый столбец, сдвинув его на ячейку вниз:
> в меню Column (операции со столбцами) выполните команду Shift Down (сдвинуть столбец вниз);
> выберите мышью столбец МАТ;
> введите 1 в ответ на вопрос о количестве ячеек, на которое сдвигается столбец (Number of rows to shift down);
> в окне Column Name (имя столбца) введите имя нового столбца, например, МАТ-1.
При прогнозировании часто используют процедуру сглаживания исходного временного ряда. Это позволяет выделить тренд — регулярную составляющую временного ряда. При проведении этой процедуры полезным может оказаться знание периода исходного временного ряда, если он, конечно, есть. В наших данных явного периода нет. В таких случаях число уровней ряда, по которым производится сглаживание, выбирается в зависимости от природы значений ряда и от степени предполагаемой его зашумленности помехой. Обычно это число лежит в интервале от 3 до 7. Для рассматриваемого случая можно выбрать 4 уровня ряда.
Проведите сглаживание исходного ряда значений МАТ методом скользящего среднего:
>в меню Operate выберите MOVING AVERAGE (скользящее среднее);
> в появившемся подменю выберите пункт Simple (обычный);
> щелкните мышью на столбце МАТ;
> система запрашивает число уровней ряда для сглаживания (Number of rows to average). Введите 4;
> система запрашивает имя нового столбца (Label the new column).
Введите, например, Маvrg.
Теперь желательно воспользоваться данными о приведенной среднегодовой температуре поверхностного слоя атмосферы Земли (столбец RAAT) и числами Вольфа (столбец WN). Для этого необходимо провести корреляционный анализ временных рядов, который позволит установить, связаны ли эти величины с прогнозируемым значением. В процессе корреляционного анализа определяется, согласованы ли два временных ряда. Проверяется, уменьшается ли один ряд с увеличением другого, изменяются ли они синхронно.
Замечание. В рассматриваемой свободно распространяемой версии BrainMaker нет доступа к функции проведения корреляционного анализа, в отличие от платной профессиональной версии (professional).
Результаты корреляционного анализа временных рядов WN и МАТ+1 дают следующее: при возрастании числа Вольфа (WN) следует ожидать уменьшения МАТ+1 через 4 года. Очевидно, что с исходным рядом МАТ подобные изменения будут происходить соответственно не через 4, а через 5 лет. Т. е. через пять лет после возрастания числа Вольфа вероятно понижение среднегодовой температуры в Мюнхене.
Таким образом, можно использовать WN для прогнозирования МАТ. Несмещенный столбец чисел Вольфа уже есть. Осталось добавить к исходным данным колонку значений \\ТМ четырехлетней давности.
Сдвиньте колонку WN на четыре ячейки вниз:

> выберите мышью столбец WN;

> в окне Column Name (имя столбца) введите имя нового столбца, например, WN-4.
Аналогично корреляционный анализ показывает, что вероятна связь между прогнозируемым значением среднегодовой температуры в Мюнхене (МАТ+ 1) и значением приведенной среднегодовой температуры поверхностного слоя атмосферы Земли (RААТ) семилетней давности. Используем это для составления базы примеров.

Задача следующего этапа- определение набора величин, которые должны будут подаваться на вход нейросети, чтобы её выход предсказывал МАТ+1. В одной из таких величин должно быть сглаженное значение МАТ (столбец Mavrg), оно характеризует тренд- тенденцию изменения МАТ. Включим во входной вектор историю последних изменений среднегодовой температуры (столбцы МАТ и МАТ-1). Необходимо также использовать величины, связанные с искомым прогнозным значением: числа Вольфа (столбец WN), числа Вольфа четырехлетней давности (WN-4) и приведенную среднегодовую температуру поверхностного слоя атмосферы Земли семилетней давности (RААТ-7).
Сформируем базу примеров — множество пар вида:
{(Mavrg_i,MAT_i,(MAT-1)_i,(WN-4)_i,(RAAT-7)_i,(MAT+1)_i ) },
где (Mavrg_i,MAT_i,(MAT-1)_i,(WN-4)_i,(RAAT-7)_i,(MAT+1)_i ) — входной вектор нейросети; (МАТ+ 1) — эталонный результат; i — обозначение одного из годов, для которого есть исходные данные; (Mavrg_i,MAT_i,(MAT-1)_i,(WN-4)_i,(RAAT-7)_i ) — значения величин в i-й году.
В соответствии с таким выбором элементов входного вектора чтобы определить значение МАТ+1 за 1973 год, нейросеть будет использовать значения Маvrg, МАТ и WN за этот же год, значение МАТ за 1972 год, значение WN за 1969 год и значение RААТ за 1966 год.
Обратите внимание, что все требуемые значения находятся на одной строке — это требование №[Макег. Поэтому мы сдвигали значения столбцов. Продолжим подготовку примеров.

На компьютере с процессором Pentium с тактовой частотой 200 МГц процесс обучения данной сети занимает от 4 до 12 минут. Для настройки межнейронных весов используются случайные числа, поэтому точное время указать сложно.
Тестирование сети
Протестируйте обученную сеть:
> в меню Operate (действия) выполните команду Test Network (тестировать сеть).
Результаты тестирования показаны во второй строке окна (рис. 8).
Сеть тестировалась 5 примерами. На 4 из них она дала правильный ответ, в одном примере она ошиблась. Для такого небольшого объема данных для обучения это —хороший результат.
Уберите с экрана окно Network Progress:
> в меню Display (отображение) выберите пункт Network Progress Display (отображение динамики обучения).

Рис. 9. Изменение формата представления
Измените формат представления данных с горизонтальных полосок на числа:
> в меню Display выполните команду Edit Network Display… (изменить формат представления);
> в появившемся окне (рис. 9) замените везде формат Termometr (полоска) на Number (число). Нужный формат выбирается из выпадающего списка;
> нажмите ОК.
Сеть готова выдать прогноз.
